1 @page page_guide_docker_cc Docker: Cross Compiling Container
2 
3 Describes how to use the "cross_compile" helper container.
4 
5 
6 Leosac comes with a container tailored for cross compiling Leosac to
7 RaspberryPi.
8 The process of cross-compilation bundles dependencies and build them.
9 
10 Requirements
11 ------------
12 
13 The container must first be built. Building the container *does not* build
14 Leosac. Instead it builds and install dependencies and required toolchain.
15 
16 You must first build the "cross_compile" container before cross-compiling
17 Leosac. The easiest way to do: `leosaccli dev docker build cross_compile`.
18 
19 How it works
20 -------------
21 
22 When building the container, required library are built and installed
23 into `/opt/rpi_fakeroot` in the container.
24 
25 Running the container will build an ARM version Leosac. For this, the container
26 expects two or three volumes:
27  + /leosac : the root of the Leosac repository (read-only).
28  + /leosac_arm_build : Build output directory (read-write).
29  + /ssh_deploy_key : An optional volume that points to a valid ssh private key.
30  This is used by command `leosaccli dev cc dev-push` command.
31 
32 The build will take place in `/leosac_arm_build`.
33 
34 How to use
35 -----------
36 
37 The recommended way to interact with the cross-compile container is to use
38 `leosaccli dev cc XXX`.
39 
40 Run `leosaccli dev cc --help` to see what you can do.
41 
42 Example
43 -------
44 
45 Run the following:
46  + `leosaccli dev cc cmake`
47  + `leosaccli dev cc fakeroot-package`
48  + `leosaccli dev cc make`
49  + `leosaccli dev cc package`
50 
51 This will give you a `fakeroot.tar` that you should extract on your
52 RaspberryPi (at `/opt/rpi_fakeroot`). Similarly, you should upload your `LEOSAC-x.y.x.sh` and
53 install it on the raspberry.
54 
55 Assuming you extract `fakeroot.tar` to `/opt/rpi_fakeroot`.
56 Now, to run Leosac, you must export `LD_LIBRARY_PATH` so that it points
57 to `/opt/rpi_fakeroot/lib`.
