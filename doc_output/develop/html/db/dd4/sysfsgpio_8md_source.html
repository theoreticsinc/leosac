10 Introduction {#mod_sysfsgpio_intro}
11 ===================================
12 
13 Welcome to the main documentation page for the SysFsGpio module.
14 This page is about SysFsGpio from an end-user point of view. \n
15 It describes how to configure the SysFsGpio module for Leosac.
16 
17 This module provide support for FGPIO device by supporting the
18 Linux Kernel SysFs interface. It allows the application to deals with
19 GPIO pin that are in fact controlled through sysfs.

21 Configuration Options {#mod_sysfsgpio_user_config}
22 ==================================================
23 
24 Below are the configuration options available.
25 
26 Options | Options | Options | Description | Mandatory
27 --------|---------|----------------|-----------------------------------------------------------------------------------------|-----------
28 aliases | | | Define GPIO aliases. This is useful to support multiple platform | **YES**
29 ---> | default | | Default name resolution for pin. `__NO__` will be replace by the `no` field | NO
30 ---> | PIN_ID | | Option name shall be the pin number, **not** textual `PIN_ID`. Value is the identifier for the pin. | NO
31 export_path | | | Absolute path to "export" sysfs file | **YES**
32 unexport_path | | | Absolute path to "unexport" sysfs file | **YES**
33 value_path | | | Absolute path to "value" file. `__PLACEHOLDER__` shall act as a placeholder for pin identifier. | **YES**
34 edge_path | | | Absolute path to "edge" file. `__PLACEHOLDER__` shall act as a placeholder for pin identifier. | **YES**
35 direction_path | | | Absolute path to "direction" file. `__PLACEHOLDER__` shall act as a placeholder for pin identifier | **YES**
36 gpios | | | List of GPIOs pins we configure | **YES**
37 ---> | gpio | | Configuration informations for one GPIO pin. | **YES**
38 ---> | ---> | name | Name of the GPIO pin | **YES**
39 ---> | ---> | no | Number of the GPIO pin. | **YES**
40 ---> | ---> | direction | Direction of the pin. This in either `in` or `out` | **YES**
41 ---> | ---> | interrupt_mode | What interrupt do we care about? See below for details | NO
42 ---> | ---> | value | Default value of the PIN. Either `1` or `0` | NO

44 Path information
45 ----------------
46 Path configuration allows the user the use the same `sysfsgpio` module on multiple platform even when
47 the path-to-gpio / naming-convention of GPIO pins varies.
48 The `__PLACEHOLDER__` placeholder will be replaced by the *identifier* of the pin.
49 The *identifier* is computed through aliases resolution.
50 
51 A simple example:
52  + Pin id / ping number is `14`.
53  + `value_path` = `/sys/class/gpio/__PLACEHOLDER__/value`.
54  + `aliases.default` = `gpio__NO__`.
55  + The pin *identifier* is resolved as `gpio14` (since `__NO__` from the default alias is replace
56  by `14`).
57  + `__PLACEHOLDER__` is replaced by the pin identifier: The module will resolve
58  the value path of the PIN to `/sys/class/gpio/gpio14/value`
59 
60 @see SysFsGpioConfigTest for more example.
61 
62 Interrupt Mode
63 --------------
64 The configuration option `interrupt_mode` can take 4 values:
65  + `Falling`
66  + `Rising`
67  + `Both`
68  + `None`. This is the default.
69 This parameter is ignored for output pin.
70 
71 Default Value
72 -------------
73 The default value (ignored for input pin) is set for the pin when the module
74 starts. It is also restored when the module stops.

77 Example {#mod_sysfsgpio_example}
78 --------------------------------
79 
80 This is a example of SysFsGpio possible configuration for SysFsGpio module into Leosac:
81 
82 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~.xml
83  <module>
84  <name>SYSFS-GPIO</name>
85  <file>libsysfsgpio.so</file>
86  <level>2</level>
87  <module_config>
88  <aliases>
89  <default>gpio__NO__</default>
90  </aliases>
91  <export_path>/sys/class/gpio/export</export_path>
92  <unexport_path>/sys/class/gpio/unexport</unexport_path>
93  <edge_path>/sys/class/gpio/__PLACEHOLDER__/edge</edge_path>
94  <value_path>/sys/class/gpio/__PLACEHOLDER__/value</value_path>
95  <direction_path>/sys/class/gpio/__PLACEHOLDER__/direction</direction_path>
96  <gpios>
97  <gpio>
98  <name>wiegand_data_high</name>
99  <no>15</no>
100  <direction>in</direction>
101  <interrupt_mode>falling</interrupt_mode>
102  </gpio>
103  <gpio>
104  <name>wiegand_data_low</name>
105  <no>14</no>
106  <direction>in</direction>
107  <interrupt_mode>falling</interrupt_mode>
108  </gpio>
109  <gpio>
110  <name>a_random_gpio</name>
111  <no>3</no>
112  <direction>out</direction>
113  <value>0</value>
114  </gpio>
115  </gpios>
116  </module_config>
117  </module>
118 ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~
