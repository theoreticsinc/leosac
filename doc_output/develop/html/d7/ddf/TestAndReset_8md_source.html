8 Introduction {#mod_testandreset_intro}
9 ======================================
10 
11 The TestAndReset module allows end user to have visual and/or audio feedback when
12 presenting a test card.
13 It also allow the user to reset Leosac configuration to its factory default.
14 
15 @note This module is only able to watch module that sends SIMPLE_WIEGAND as auth-source
16  data format.
17 
18 Configuration Options {#mod_testandreset_user_config}
19 =====================================================
20 
21 This is a module that watch input device and check for preconfigured reset/restart card.
22 Options | Options | Options | Description | Mandatory
23 --------------|-----------|------------|----------------------------------------------------------------|-----------
24 test_buzzer | | | Device (beeper) to control when test card is read | NO
25 test_led | | | Device (led) to control when test card is read | NO
26 run_on_start | | | Should we beep/blink when the module starts ? | NO (defaults to true)
27 promisc | | | Listen to all event to watch for reset card. | NO (defaults to false)
28 reset_card | | | Wiegand ID of reset card. Only useful if promisc is `true`. | YES if promisc is `true`.
29 devices | | | List of devices to watch for | NO
30 ---> | device | | Watch for event sent by this device | YES
31 ---> | ---> | name | Name of the device | YES
32 ---> | ---> | reset_card | Content / number of the card | NO
33 ---> | ---> | test_card | Id of the card that trigger test led/beep stuff | NO
34 
35 About `promisc` mode: It will listen to everything on the message bus to try and detect the reset card.
36 This comes at a little performance cost because we watch way more message than needed. However, this
37 has the advantage of working without specifying a source device, which may be helpful when
38 synchronizing configuration.
